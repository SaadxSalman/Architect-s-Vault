# Para-Master: Multimodal Fine-Tuning for Parasite Identification ğŸ¦ ğŸ”

Building on previous work with vision models and embeddings, **Para-Master** is a specialized, highly accurate model for identifying parasites from various sources. It is fine-tuned on a vast dataset of microscopic images of blood smears, tissue samples, and fecal samples to classify different parasite species and stages with unprecedented accuracy.

-----

## âœ¨ Features

  * **Highly Accurate Classification:** The fine-tuned **Vision Transformer** can recognize subtle morphological features of parasites that are often difficult for the human eye to detect.
  * **Multimodal Data Analysis:** Analyzes both visual data (microscopic images) and genomic data (DNA/protein sequences) for a comprehensive identification.
  * **Genomic Identification:** The custom **vector embeddings model** for parasite genomics enables rapid comparison and identification of novel or mutated parasite strains.
  * **High-Quality Data Curation:** Leverages **Argilla** to ensure the massive dataset of microscopic images is accurately curated and labeled, leading to a robust and reliable model.
  * **Fast Genomic Search:** Uses **Milvus**, a high-performance vector database, to enable ultra-fast searching of genomic embeddings.

## ğŸ› ï¸ Para-Master: The Final Detailed Stack

### **1. Frontend Ecosystem**

* **User Identification Interface:** [SvelteKit](https://kit.svelte.dev/)
* *Why:* Extremely fast, minimal boilerplate, and excellent for real-time reactivity when displaying ML results.


* **Management Dashboard:** [Next.js 14+](https://nextjs.org/) (App Router)
* *Why:* Industry standard for administrative tools, robust SEO (if needed), and native support for server-side MongoDB interactions.


* **Styling:** [Tailwind CSS](https://tailwindcss.com/)
* *Why:* Rapid UI development with a consistent design system across both frontends.


* **State Management & Fetching:** [TanStack Query](https://tanstack.com/query/latest) (for Next.js) and Svelte's native stores.

---

### **2. Backend & Intelligence**

* **Primary ML API:** [FastAPI](https://fastapi.tiangolo.com/) (Python 3.10+)
* *Why:* Native support for asynchronous processing, which is critical when waiting for GPU-bound model inference or Milvus queries.


* **Service API:** [Node.js](https://nodejs.org/) with [Express](https://expressjs.com/) (within the Next.js API layer)
* *Why:* Standard MERN logic for handling metadata CRUD and authentication.


* **Data Curation:** [Argilla](https://www.argilla.io/)
* *Why:* Specifically designed for "human-in-the-loop" workflows. It allows you to refine your parasite labels before re-training.



---

### **3. AI / Machine Learning Models**

* **Vision Model:** Fine-tuned [Swin Transformer](https://github.com/microsoft/Swin-Transformer)
* *Why:* Better than standard CNNs at capturing hierarchical spatial features in microscopic slides.


* **Genomic Model:** [DNABERT](https://github.com/jerryji1993/DNABERT) or [ESM-2](https://github.com/facebookresearch/esm)
* *Why:* Converts raw DNA or protein sequences into 768-dimension vectors for similarity search.


* **Framework:** [PyTorch](https://pytorch.org/) & [Hugging Face Transformers](https://huggingface.co/docs/transformers/index).

---

### **4. Database & Infrastructure**

* **Vector Database:** [Milvus](https://milvus.io/)
* *Why:* Engineered for billion-scale vector search; it handles your genomic embeddings with sub-second latency.


* **Metadata Database:** [MongoDB](https://www.mongodb.com/)
* *Why:* Document-based storage is perfect for the varied and evolving metadata associated with different parasite species.


* **Object Storage:** [MinIO](https://min.io/) (via Docker)
* *Why:* S3-compatible storage for the raw microscopic images.


* **Containerization:** [Docker](https://www.docker.com/) & [Docker Compose](https://docs.docker.com/compose/).

---

### **5. DevOps & Quality**

* **Automation:** [GitHub Actions](https://github.com/features/actions)
* **Version Control:** Git with [LFS (Large File Storage)](https://git-lfs.com/) for model weights.
* **API Documentation:** [Swagger UI](https://swagger.io/) (auto-generated by FastAPI).

## ğŸš€ Getting Started

### Prerequisites

  * Python 3.10+
  * Node.js (for SvelteKit)
  * Docker (for Milvus)

### Installation

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/saadsalmanakram/Para-Master.git
    cd Para-Master
    ```
2.  **Start Docker containers:**
    ```bash
    docker-compose up -d
    ```
3.  **Set up the Python backend:**
    ```bash
    cd backend
    pip install -r requirements.txt
    ```
4.  **Set up the frontend:**
    ```bash
    cd frontend
    npm install
    ```

### Configuration

Follow the instructions in the `config/` directory to connect to your Milvus instance and set up the model paths.

### Usage

Run the Python backend and the SvelteKit frontend to begin the parasite identification process.
